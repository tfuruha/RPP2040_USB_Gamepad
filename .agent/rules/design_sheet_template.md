# 設計書テンプレート

今回実装（または修正）した機能について、今後の開発でAIが迷わないための「コンテキスト資料」として設計書を作成する際のテンプレートです。

## 1. 概要と背景
- **目的**: 何のための機能か。
- **設計根拠 (Rationale)**: なぜ標準的な実装ではなく、この設計・手法を選んだのか。
- **解決した課題**: 開発中に発生したトラブルや制約と、その回避策。

## 2. インターフェースと依存関係
- **外部依存**: `#include` やインポートが必要なライブラリ・ファイル。
- **データ構造**: 主要な構造体（struct）、列挙型（enum）、クラス定義の抜粋。
- **API定義**: 主要な関数・メソッドのシグネチャ（引数と戻り値の説明）。

## 3. 設計上の重要ポイント（AIの判断基準）
- **技術的根拠**: 数式、定数、アルゴリズムの選定理由。
- **環境制約**: メモリ制限、タイミング制約、ハードウェア特有の挙動。
- **構成管理**: マジックナンバーの排除方法や、フラグによる管理構造。

## 4. 実装原則・禁止事項
- **手順**: 「必ずこの順序で実行すること」といったシーケンスの制約。
- **禁止事項**: 「この変数は直接操作しない」「このライブラリ関数は使用禁止」等の制約。

## 5. 検証方法
- **動作確認手順**: 正常動作を確認するためのテスト方法や期待される出力。
- **エラー確認**: 例外系や異常系が正しく処理されているかの確認方法。

---
## 執筆時のスタイル指示
- AIが今後のコーディングで「唯一の正解（Source of Truth）」として参照できるよう、論理的かつ厳密なトーンで記述すること。
- 曖昧な表現（〜だと思われる等）は避け、断定的に記述すること。
